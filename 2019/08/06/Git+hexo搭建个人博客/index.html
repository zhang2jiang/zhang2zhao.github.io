前言

这是一篇自己关于如何使用GitHub+Hexo搭建个人博客的总结。在机缘巧合之下知道可以利用GitHub作为一个免费的服务器，为我们建立个人博客提供很大的方便，免去自建服务器和写数据库的不便。

同时，Hexo是一个快速、简洁高效的博客框架，使用Markdown格式来编写文件，与GitHub相结合可以很快的生成静态网页。

安装Git

点击此处访问官网，按需下载相应版本，默认安装即可
参考资料：《如何在windows下安装GIT》 　（By 俊雨廷休）

Git客户端实际上十分强大，它本身就可以的创建本地仓库，而本地仓库和远程仓库之间的同步也是通过Git客户端完成的。Git是版本管理工具，可以学习Git中文教程。若安装成功，在桌面右键鼠标，可以看到菜单里多了 Git GUI Here’和 Git Bash Here两个选项，第一个是图形界面的Git操作，另一个是命令行。利用命令行检查是否安装成功。

安装node.js

我们需要知道安装hexo要用到npm,而安装npm则需要node.js,点击此处访问官网，按需下载相应版本，默认安装可以了，然后通过命令行来判断是否安装成功。

npm可以设置成淘宝NPM镜像来提高下载速度。

将博客部署到 Github Pages 上

到目前为止，我们的本地博客就成功搭建了，但只能通过本地连接查看博客，我们要做的是让其他人也能够访问我们的博客，这就需要我们将博客部署到Github Pages上。

GitHub相关

帐号注册

在创建博客之前，先要有GitHub的帐号，该帐号将用来创建项目，默认的域名username.github.io/projectName中的username也要用到这个帐号。
注意：下面涉及到的一些命令凡是更用户名和项目名有关的一律会用这里的username和projectName代替，注意替换。
访问：http://www.github.com/ sign up for free的意思就是“免费注册登录”，注册你的username和邮箱，邮箱十分重要，GitHub上很多通知都是通过邮箱的。比如你的主页上传并构建成功会通过邮箱通知，更重要的是，如果构建失败的话也会在邮件中说明原因。
